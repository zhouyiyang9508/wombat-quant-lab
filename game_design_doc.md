# 🎮 Roguelike 游戏事件系统设计文档 (Project: Wombat Spire)

## 1. 概述
本项目旨在开发一个类《杀戮尖塔》(Slay the Spire) 的 Roguelike 游戏原型，核心特色在于“正派 (Orthodox)”与“邪教 (Cult)”双重路线的地图事件与数值机制。

## 2. 核心机制设计

### 2.1 派系机制 (Faction System)
- **正派 (Orthodox):** 
  - **风格**: 稳健、防御导向。
  - **加成**: 小幅增加攻击力，提供护盾或生命恢复。
  - **代价**: 消耗金币或正气值。
- **邪教 (Cult):** 
  - **风格**: 极端、进攻导向、高风险高回报。
  - **加成**: 巨额增加攻击力 (Attack Up)，显著降低防御力 (Defense Down)。
  - **代价**: 扣除生命上限、获得永久负面诅咒、随机烧毁手牌。

### 2.2 地图事件 (Map Events)
- 地图节点随机生成“正派”或“邪教”标记。
- **正派事件示例**: [金光咒] - 防御+5，生命恢复10%。
- **邪教事件示例**: [血祭仪式] - 攻击力翻倍，本局战斗防御力永久-30%，失去10点生命上限。

### 2.3 UI 视觉规范 (UI/UX)
- **血条 (Health Bar)**: 位于角色下方，包含数值显示。
- **护甲 (Block)**: 血条左侧蓝色盾牌图标及数值。
- **状态图标 (Buff/Debuff)**: 血条下方排列，悬停显示详细说明。
- **派系特效**: 
  - 正派：金色、青色微光。
  - 邪教：暗红、紫色、破碎的边框特效。

## 3. 技术栈建议
- **框架**: Python + Pygame (快速原型) 或 Go + Ebitengine。
- **AI 角色**: 
  - `GameMasterAgent` (VeADK-Go): 负责根据派系实时生成平衡的数值和文本。
  - `CodeBear`: 负责具体的工程实现。
